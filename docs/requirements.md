# Requirements Document - IAM Super

## 1. Functional Requirements (FR)

| ID   | Description                                                                                                    | Actor         |
|:-----|:---------------------------------------------------------------------------------------------------------------|:--------------|
| FR01 | The system must allow administrators to register new sectors/departments.                                      | Admin         |
| FR02 | The system must allow administrators to register new processes.                                                | Admin         |
| FR03 | The system must allow administrators to assign processes to specific sectors.                                  | Admin         |
| FR04 | The system must allow administrators to register new standard users.                                           | Admin         |
| FR05 | The system must allow standard users to access information regarding the processes of their assigned sector.   | Standard User |
| FR06 | The system must generate a formatted message containing all user access links/info upon onboarding completion. | System        |
| FR07 | The system must allow administrators to edit user information.                                                 | Admin         |
| FR08 | The system must allow administrators to deactivate users.                                                      | Admin         |
| FR09 | The system must allow users to log in and log out.                                                             | System        |
| FR10 | The system must allow users to recover their login passwords.                                                  | System        |
| FR11 | The system must maintain an audit log tracking who modified specific user permissions and when.                | System        |

## 2. Non-Functional Requirements (NFR)

| ID    | Category     | Description                                                                                       |
|:------|:-------------|:--------------------------------------------------------------------------------------------------|
| NFR01 | Security     | The system must authenticate users via JWT (JSON Web Token) generated by the backend.             |
| NFR02 | Availability | The system must be accessible via web and feature a responsive interface for mobile devices.      |
| NFR03 | Performance  | User listing must be completed within 2 seconds for a volume of up to 1,000 users.                |
| NFR04 | Security     | Passwords must be securely stored using the BCrypt hashing algorithm.                             |
| NFR05 | Integrity    | Database updates involving multiple tables must ensure atomicity through the use of transactions. |
